# Правила проекта (Conventions)

## Структура репозитория

- `/backend` — Go backend (REST API + бизнес-логика + БД)
- `/frontend` — JS frontend (веб-интерфейс)
- `/docs` — документация (контракт API, правила, решения)

## Процесс работы (ветки и Pull Request)

Главная ветка:

- `main` всегда должна быть рабочей
- в `main` напрямую не пушим — только через Pull Request (PR)

Имена веток:

- `feat/<область>-<коротко>` — новая функциональность
- `fix/<область>-<коротко>` — исправление бага
- `docs/<коротко>` — документация
- `chore/<коротко>` — технические изменения (настройки, рефакторинг)

Области (рекомендуемые):

- backend, frontend, docs, infra

Примеры:

- `feat/backend-auth`
- `feat/frontend-dashboard`
- `fix/backend-import-dedup`
- `docs/api-v0`

Правила PR:

- один PR = одна задача (одна фича/фикс)
- PR должен быть небольшим и понятным для ревью
- в описании PR обязательно:
  1. что сделано
  2. как проверить локально
  3. (для фронта) скриншоты, если менялся UI

## Сообщения коммитов

Желательно так:

- `feat: ...`
- `fix: ...`
- `docs: ...`
- `chore: ...`

Примеры:

- `feat: добавить фильтры транзакций`
- `fix: корректно обрабатывать пустые строки csv`
- `docs: обновить контракт api для бюджетов`

## Правила API

- базовый префикс: `/api`
- все ответы — JSON
- формат ошибок единый: `{ "error": "сообщение" }`

### Даты и время

- timestamp (created_at и т.п.) — RFC3339 в UTC
- дата операции (occurred_at) — `YYYY-MM-DD`
- месяц в отчётах — `YYYY-MM`

### Деньги

- на бэке храним и возвращаем деньги в **центах** (целое число): `amount_cents`
- на фронте форматируем для показа (€, ₽ и т.д.)

### Коды ответов (минимум)

- 200 OK — успех
- 201 Created — создано
- 400 Bad Request — неверные данные
- 401 Unauthorized — нет/неверный токен
- 403 Forbidden — нет прав
- 404 Not Found — не найдено
- 409 Conflict — конфликт (например, дубль при импорте)
- 500 Internal Server Error — неожиданная ошибка

## Договорённости по фронту

- все запросы к API идут через один слой (например `src/api/*`)
- страницы:
  - `/login`
  - `/dashboard`
  - `/transactions`
  - `/budgets`
  - `/import`
  - `/rules`
